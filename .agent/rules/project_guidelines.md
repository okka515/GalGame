# Project Guidelines & Rules

このプロジェクトにおける開発・コード生成の基本ルールです。AIアシスタントなどのツールを利用する際も、このルールに則って提案・実装を行ってください。

## 基本スタック・コアライブラリ
- **React + TypeScript + Vite**: 本プロジェクトのベースとなる技術スタックです。
- **NarraLeaf-React**: ビジュアルノベル・ギャルゲーのコアエンジンとして使用しています。
  - **重要**: 新機能の追加、シナリオの作成、ゲームロジックの変更を行う際は、必ず公式ドキュメントである **[NarraLeaf-React](https://react.narraleaf.com/)** を参照し、その仕様に従ってください。
  - ゲームの構築には、主に `narraleaf-react` が提供する `Scene`, `Menu`, `Game`, `Player` などのAPIを使用します。

## ディレクトリ構成と規約
現在のプロジェクト構造に基づき、以下の役割に沿ってファイルを配置・編集してください。

- `src/scenarios/`
  - シナリオやイベントの定義を行います。
  - 新しいシーンは `new Scene("シーン名")` で作成し、`.action()` で進行を記述します。
  - 共通ルートや各チャプターごとにファイルを分割し、`scene.jumpTo()` で繋ぐ設計にしています。
- `src/characters/`
  - キャラクターの定義（名前、立ち絵設定など）を配置します。
  - シナリオ内ではここで定義されたキャラクターオブジェクト（例: `haruchiro.say("...")`）を使用します。
- `src/game.ts`
  - `new Game()` によるゲームインスタンスの初期化と設定（解像度など）を行う中心ファイルです。
- `src/App.tsx`
  - Reactのエントリポイントです。`<GameProviders>` と `<Player>` を用いて描画エンジンをマウントします。

## コーディングスタイル
- 原則として **TypeScript** を使用し、型安全な実装を心がけてください。
- NarraLeaf-React のアクションやメニュー構築（例: `Menu.prompt().choose()`）は、可読性の高いメソッドチェーンを活用してください。
- ファイル間の依存関係を整理し、関心の分離（シナリオ定義とゲームシステム設定の分離など）を保ってください。
